CHAPTER 1 - INTRODUCTION

Competitive Programming Handbook (diễn giải đầy đủ)

1. Competitive Programming là gì?

Competitive Programming là sự kết hợp giữa thiết kế thuật toán và lập trình thực thi. Mục tiêu là giải bài toán nhanh, đúng và tối ưu.

Một lời giải tốt không chỉ cần đúng mà còn phải chạy trong thời gian giới hạn.

2. Hai kỹ năng quan trọng

🔹 Thiết kế thuật toán

  • Phân tích bài toán
  • Tìm hướng giải hiệu quả
  • Áp dụng tư duy toán + logic

🔹 Cài đặt thuật toán

  • Code đúng logic
  • Không lỗi runtime
  • Tối ưu thời gian chạy

3. Ngôn ngữ lập trình

Ba ngôn ngữ phổ biến nhất trong thi đấu:

  • C++: nhanh nhất, nhiều thư viện → phổ biến nhất
  • Python: dễ viết, mạnh xử lý số lớn
  • Java: ổn định, dùng ít hơn
C++ gần như là tiêu chuẩn vì tốc độ và STL rất mạnh.

4. Template C++ thi đấu 

#include <bits/stdc++.h>
using namespace std;
                    
int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    // code
    return 0;
}

Template này giúp code nhanh và tối ưu input/output.

5. Input / Output

  • Dữ liệu nhập qua cin, xuất qua cout
  • Dữ liệu cách nhau bằng space hoặc newline
  • Có thể đọc đến khi hết input
Tối ưu tốc độ: 
ios::sync_with_stdio(false);
cin.tie(nullptr);

6. Kiểu dữ liệu số

🔹 Integer

  • int: ~2e9
  • long long: ~9e18
Sai lầm phổ biến: nhân int gây tràn số dù gán vào long long.

🔹 Số thực

  • double: phổ biến
  • long double: chính xác hơn

So sánh số thực không dùng == mà dùng sai số epsilon.

7. Modulo (chia dư)

Trong lập trình thi đấu, kết quả thường rất lớn nên ta lấy modulo (ví dụ 1e9+7).

  • (a + b) % m
  • (a * b) % m

Ta luôn có thể lấy mod sau mỗi phép tính để tránh tràn số.

8. Tối giản code

Trong thi đấu, code ngắn giúp viết nhanh hơn.

  • typedef / using để rút gọn kiểu dữ liệu
  • macro để viết nhanh vòng lặp
Ví dụ: 
#define F first
#define S second
#define PB push_back
#define MP make_pair

9. Tư duy toán học

Competitive Programming yêu cầu nền tảng toán học tốt.

  • Công thức tổng
  • Cấp số cộng / nhân
  • Logarithm
  • Logic & tập hợp

10. Môi trường thi đấu

  • IOI: học sinh THPT
  • ICPC: sinh viên
  • Online contest: Codeforces, AtCoder...

Các cuộc thi giúp rèn luyện tư duy thuật toán và tốc độ code.

Kết luận

Competitive Programming = Thuật toán + Tư duy + Tốc độ code. Muốn giỏi cần luyện nhiều bài và hiểu bản chất thuật toán.
